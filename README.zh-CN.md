# protoc-gen-gorm

[English](README.md)

`protoc-gen-gorm` 是一个 Go 语言的 `protoc` 插件，用于根据 protobuf message 生成适配 GORM 的模型代码。它读取 `option/gorm.proto` 中定义的自定义选项，并生成 Go 结构体、GORM tag、表名方法、protobuf/model 转换方法、可选的查询 helper，以及可选的 DAO 封装。

## 功能特性

- 为设置了 `(gorm.set_table) = true` 的 protobuf message 生成 `MessageModel` 结构体。
- 生成列名、主键、索引、唯一索引、默认值、`not null`、软删除和 SQL 类型相关的 GORM tag。
- 支持通过 `(gorm.table_name)` 自定义表名。
- 支持 protobuf message 与生成的 model 结构体之间互相转换。
- 支持 timestamp、整数/字符串时间字段、枚举、repeated 字段和 map 字段的 GORM 类型映射。
- 可通过 `with_gorm_option=true` 生成通用 GORM helper 函数。
- 可通过 `with_gorm_dao=true` 生成每个 message 对应的 DAO 类型。

## 仓库结构

```text
.
├── main.go                         # protoc 插件入口
├── internal/protoc-gen-gorm/       # 生成器实现
├── option/
│   ├── gorm.proto                  # 自定义 protobuf options
│   ├── gorm.pb.go                  # 生成后的 Go option 定义
│   └── genpb.sh                    # 重新生成 option/gorm.pb.go
├── example/
│   ├── model.proto                 # 示例 protobuf 输入
│   ├── genpb.sh                    # 重新生成示例代码
│   └── model/                      # 示例生成代码和测试
└── third_party/proto/              # vendored protobuf descriptor 文件
```

## 环境要求

- Go 1.20 或更高版本
- `protoc`
- `protoc-gen-go`
- `go.mod` 中声明的 GORM 相关依赖

如需安装标准 Go protobuf 生成器：

```sh
go install google.golang.org/protobuf/cmd/protoc-gen-go@latest
```

## 构建与安装

构建本地二进制文件：

```sh
make build
```

安装插件到 Go binary 路径：

```sh
make install
```

整理依赖：

```sh
make tidy
```

## 使用方式

在 proto 文件中引入 GORM option 定义：

```proto
import "option/gorm.proto";
```

为需要生成 model 的 message 设置选项：

```proto
message User {
  option (gorm.set_table) = true;
  option (gorm.table_name) = "users";

  int32 id = 1 [(gorm.rules) = {int: {type: bigint}, primary_key: true}];
  string name = 2 [(gorm.rules) = {index: {name: "idx_name"}}];
  google.protobuf.Timestamp created_at = 3 [
    (gorm.rules) = {time: {type: datetime, auto_create_time: true}, not_null: true}
  ];
}
```

同时运行 Go 生成器和本插件：

```sh
protoc -I. \
  -I./third_party/proto \
  --go_out=. --go_opt paths=source_relative \
  --gorm_out=. --gorm_opt paths=source_relative \
  your_file.proto
```

生成额外的 helper 和 DAO 文件：

```sh
protoc -I. \
  -I./third_party/proto \
  --go_out=. --go_opt paths=source_relative \
  --gorm_out=. --gorm_opt paths=source_relative \
  --gorm_opt with_gorm_option=true \
  --gorm_opt with_gorm_dao=true \
  your_file.proto
```

`with_gorm_dao=true` 应与 `with_gorm_option=true` 一起使用。

## 生成文件

以 `model.proto` 为例，插件可以生成：

- `model.gorm.pb.go`：GORM model 结构体、tag、表名方法、转换方法，以及 repeated/map 字段使用的 JSON scanner/valuer helper。
- `model.gorm.option.pb.go`：通用 GORM options 和 helper 函数，例如 `Create`、`Save`、`First`、`Find`、`Count`、`Delete`、`Limit`、`Offset`、`Order` 和 `TableName`。
- `model.gorm.dao.pb.go`：每个 message 对应的 DAO 封装，例如带有类型化 CRUD 方法的 `UserDao`。

未设置 `(gorm.set_table) = true` 的 message 默认不会生成，除非它被已生成的 model 类型嵌入使用。

## 支持的 Options

Message options：

- `(gorm.set_table) = true`：启用该 message 的生成。
- `(gorm.table_name) = "name"`：覆盖默认 snake_case 表名。
- `(gorm.disable_snake_case)`：已在 proto options 中定义，预留给命名行为使用。

Field options：

- `(gorm.ignore_gorm_column) = true`：生成 `gorm:"-"`。
- `(gorm.rules).column_name`：覆盖生成的列名。
- `(gorm.rules).primary_key`：生成主键 tag。
- `(gorm.rules).index` 和 `(gorm.rules).uniqueIndex`：生成索引 tag。
- `(gorm.rules).not_null`：生成 `not null`。
- bool、int、float、time、binary、string 和 enum 字段都有对应的类型规则。

Repeated 字段和 map 字段会以 `longtext` 存储，并生成自定义 Go 类型，通过 JSON marshal/unmarshal helper 实现 `driver.Valuer` 和 `sql.Scanner`。

## 类型映射说明

- 带 time rules 的 `google.protobuf.Timestamp` 会映射为 `time.Time`、`datatypes.Date` 或 `gorm.DeletedAt`。
- `int64`、`uint64` 和 `string` 字段在设置 time rules 后也可以作为时间字段处理。
- enum 字段可以按 SQL `enum`、`string` 或 `int` 存储。
- 非 timestamp message 字段会以 `embeddedPrefix` 方式嵌入。
- 每个生成的 model 层级只允许一个主键字段和一个软删除字段。

## 示例

重新生成内置示例：

```sh
make build
sh example/genpb.sh
```

示例输入位于 `example/model.proto`，生成代码位于 `example/model/`。

## 测试

运行全部测试：

```sh
go test ./...
```

当前 `example/model/model.gorm.pb_test.go` 中的示例测试会连接本地 MySQL，并使用硬编码的开发凭据。如果本地数据库不可用，构建和代码生成检查仍可能通过，但示例集成测试会失败。

## 开发说明

- 修改 Go 文件后运行 `gofmt`。
- 修改 `option/gorm.proto` 后重新生成 `option/gorm.pb.go`：

```sh
sh option/genpb.sh
```

- 修改生成器行为后重新生成示例输出：

```sh
make build
sh example/genpb.sh
```

- 生成文件包含 `// Code generated by protoc-gen-gorm. DO NOT EDIT.`，通常应通过修改 proto 或生成器来更新，而不是手动编辑。
